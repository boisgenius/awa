# Claw Academy 开发计划

> 基于 Next.js 14 全栈架构，适配 Claude Code 开发方式

## 项目概述

| 项目 | 说明 |
|------|------|
| **名称** | Claw Academy |
| **仓库** | 当前仓库 (awa) |
| **定位** | AI 智能体技能市场平台 |
| **技术栈** | Next.js 14 + TypeScript + Tailwind + Supabase + Solana |
| **架构** | Next.js 全栈 (Serverless) |
| **包管理** | pnpm |
| **认证方式** | Agent API Key（无需用户钱包登录） |
| **支付方式** | Agent 绑定钱包自动支付 |
| **后端方案** | Next.js API Routes + Supabase（无独立后端服务） |

---

## 全栈架构说明

> **无需单独的后端服务**，使用 Next.js 全栈 Serverless 方案

```
┌─────────────────────────────────────────────────────────────────┐
│                    Next.js 14 App Router                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  前端 (Client)              后端 (Server)                        │
│  ┌────────────────┐        ┌────────────────────────────────┐   │
│  │ React 组件     │        │ API Routes (app/api/*)         │   │
│  │ - 页面渲染     │  ────▶ │ - REST API 端点                │   │
│  │ - 用户交互     │        │ - 请求验证                     │   │
│  │ - 状态管理     │        │ - 业务逻辑                     │   │
│  └────────────────┘        └────────────────────────────────┘   │
│                                       │                          │
│                                       ▼                          │
│                            ┌────────────────────────────────┐   │
│                            │ Server Actions                  │   │
│                            │ - 表单处理                     │   │
│                            │ - 数据变更                     │   │
│                            └────────────────────────────────┘   │
│                                       │                          │
└───────────────────────────────────────│──────────────────────────┘
                                        │
                    ┌───────────────────┼───────────────────┐
                    ▼                   ▼                   ▼
          ┌─────────────────┐ ┌─────────────────┐ ┌─────────────────┐
          │    Supabase     │ │   Solana RPC    │ │  External APIs  │
          │   (PostgreSQL)  │ │    (Helius)     │ │                 │
          │                 │ │                 │ │                 │
          │ - 数据存储      │ │ - 交易提交      │ │ - 第三方服务    │
          │ - 实时订阅      │ │ - 余额查询      │ │                 │
          │ - 文件存储      │ │ - 交易验证      │ │                 │
          └─────────────────┘ └─────────────────┘ └─────────────────┘
```

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端渲染** | Next.js App Router | SSR/SSG/CSR，React Server Components |
| **API 接口** | API Routes (`app/api/*/route.ts`) | RESTful API，处理 HTTP 请求 |
| **服务端逻辑** | Server Actions | 表单处理，数据变更，无需手写 API |
| **数据库** | Supabase (PostgreSQL) | 数据存储 + RLS 权限控制 |
| **认证** | 自实现 Agent API Key | 验证 Agent 身份 |
| **支付** | @solana/web3.js | 服务端签名，自动支付 |
| **部署** | Vercel | Serverless，自动扩展 |

## 认证与支付架构

### Agent 认证流程

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  OpenClaw Agent │────▶│  Claw Academy   │────▶│    Supabase     │
│                 │     │      API        │     │                 │
│  - API Key      │     │  - 验证 Key     │     │  - Agent 数据   │
│  - 绑定钱包     │     │  - 返回 Agent   │     │  - 钱包密钥     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### 自动支付流程

```
1. Agent 请求购买技能
2. 验证 Agent API Key
3. 获取 Agent 绑定的钱包密钥
4. 服务端自动构建并签名交易
5. 提交到 Solana 网络
6. 验证交易成功
7. 返回技能内容
```

```typescript
// 主要接口
interface AgentAuth {
  // API Key 管理
  generateApiKey(agentId: string): Promise<ApiKey>;
  validateApiKey(key: string): Promise<Agent | null>;
  revokeApiKey(keyId: string): Promise<void>;

  // Agent 钱包绑定
  getAgentWallet(agentId: string): Promise<WalletInfo>;
  hasValidWallet(agentId: string): Promise<boolean>;
}

interface Agent {
  id: string;
  name: string;
  apiKeyHash: string;
  walletPublicKey: string;      // 绑定的钱包公钥
  walletEncryptedKey: string;   // 加密存储的私钥
  createdAt: Date;
}
```

## Claude Code 开发最佳实践

### 1. 模块开发顺序 (TDD)

每个模块严格按照 **TDD (Red-Green-Refactor)** 顺序开发：

```
1. types.ts           → 先定义类型接口
2. *.test.ts (Red)    → 编写测试用例（预期失败）
3. core/*.ts (Green)  → 最小实现使测试通过
4. adapters/*.ts      → 数据库适配
5. refactor           → 重构，保持测试通过
6. index.ts           → 导出接口
```

**TDD 循环示例**:
```bash
# 1. 定义类型
> 为 src/lib/auth 定义 Agent 和 ApiKey 类型

> 实现 validateApiKey 使测试通过
pnpm test:watch auth  # ✅ 通过

# 4. 重构 (Refactor)
> 重构 validateApiKey，提取公共逻辑
pnpm test:watch auth  # ✅ 保持通过
```
